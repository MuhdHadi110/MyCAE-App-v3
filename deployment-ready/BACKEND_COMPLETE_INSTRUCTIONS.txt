================================================================================
        COMPLETE BACKEND PACKAGE - Installation Instructions
================================================================================

This package includes EVERYTHING you need - no npm install required!

Package: backend-complete-with-modules.zip (22 MB)

Contains:
  ✓ dist/ folder (compiled backend code)
  ✓ node_modules/ folder (ALL dependencies pre-installed)
  ✓ package.json
  ✓ package-lock.json

================================================================================
INSTALLATION STEPS
================================================================================

Step 1: BACKUP & REMOVE OLD BACKEND
------------------------------------
1. In cPanel File Manager, navigate to /home/mycaet40/
2. Right-click on the OLD "mycaetracker-backend" folder
3. Select "Rename" and rename it to "mycaetracker-backend-OLD"
   (This keeps a backup just in case)

Step 2: UPLOAD NEW BACKEND PACKAGE
------------------------------------
1. In File Manager, make sure you're in /home/mycaet40/
2. Click "Upload" button
3. Select: backend-complete-with-modules.zip (22 MB)
4. Wait for upload to complete (may take 2-3 minutes)

Step 3: EXTRACT THE PACKAGE
------------------------------------
1. After upload, you'll see backend-complete-with-modules.zip in the file list
2. Right-click on the ZIP file
3. Select "Extract"
4. It will create 4 items:
   - dist/
   - node_modules/
   - package.json
   - package-lock.json

Step 4: CREATE NEW BACKEND FOLDER
------------------------------------
1. Click "+ Folder" button
2. Name it: mycaetracker-backend
3. Double-click to enter the folder
4. Select all 4 items (dist/, node_modules/, package.json, package-lock.json)
5. Right-click and select "Move"
6. Move them INTO the mycaetracker-backend folder
7. After moving, delete the empty backend-complete-with-modules.zip file

ALTERNATIVE (Simpler):
------------------------------------
1. Create a NEW folder first: "mycaetracker-backend-new"
2. Enter that folder
3. Upload backend-complete-with-modules.zip directly into it
4. Extract it there
5. Delete the ZIP file
6. Go back to /home/mycaet40/
7. Delete the old "mycaetracker-backend" folder
8. Rename "mycaetracker-backend-new" to "mycaetracker-backend"

Step 5: CREATE .env FILE
------------------------------------
You still need to create the .env file:

1. In File Manager, go to /home/mycaet40/mycaetracker-backend/
2. Click "+ File"
3. Name it: .env
4. Right-click and "Edit"
5. Copy the contents from .env.production-template (from deployment-ready folder)
6. Fill in your database credentials
7. Save the file
8. Right-click .env → Change Permissions → Set to 600

Step 6: UPDATE NODE.JS APP CONFIGURATION
------------------------------------
1. Go to cPanel → Setup Node.js App
2. If you see your old app, click "Edit" or "Remove" then "Create Application"
3. Configure:
   - Node.js version: 20.19.4 (or latest LTS)
   - Application mode: Production
   - Application root: mycaetracker-backend
   - Application URL: mycaetech.com
   - Application startup file: dist/server.js
4. Click "Create" or "Update"

Step 7: SKIP NPM INSTALL!
------------------------------------
DO NOT click "Run NPM Install" - you already have node_modules!

Just click "Restart" or "Start" to start the application.

Step 8: VERIFY
------------------------------------
1. Check that status shows "Running" (green)
2. Visit: https://mycaetech.com/api/health
3. Try logging in at: https://mycaetech.com

================================================================================
TROUBLESHOOTING
================================================================================

If you see "Cannot find module" error:
  → The node_modules folder wasn't extracted properly
  → Go back and make sure you extracted the ZIP file
  → Verify node_modules/ exists in /home/mycaet40/mycaetracker-backend/

If backend won't start:
  → Check that .env file exists and has correct database credentials
  → Verify Application startup file is set to: dist/server.js
  → Check logs in Node.js App interface

If you get 503 errors:
  → Make sure the Node.js app is running (green status)
  → Verify the API .htaccess file exists in /public_html/api/
  → Restart the Node.js application
